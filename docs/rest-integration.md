# REST Integration

`ZeroAlloc.Rest` can consume `ISerializer<T>` via the `RestSerializerAdapter<T>` bridge class.

## Setup

```bash
dotnet add package ZeroAlloc.Rest
dotnet add package ZeroAlloc.Serialisation
dotnet add package ZeroAlloc.Serialisation.MemoryPack  # or your chosen backend
dotnet add package ZeroAlloc.Serialisation.Generator
```

## Usage

```csharp
// Register the ISerializer<T> for your type
services.AddOrderCreatedSerializer();  // generated by ZeroAlloc.Serialisation.Generator

// Bridge it to IRestSerializer
services.AddSingleton<IRestSerializer>(sp =>
    new RestSerializerAdapter<OrderCreated>(
        sp.GetRequiredService<ISerializer<OrderCreated>>()));
```

## How It Works

`RestSerializerAdapter<T>` implements `IRestSerializer` and delegates to `ISerializer<T>`:

- **SerializeAsync**: rents an `ArrayBufferWriter<byte>`, calls `ISerializer<T>.Serialize`, then writes `WrittenMemory` to the stream — no intermediate `byte[]` allocation
- **DeserializeAsync**: reads all bytes from the stream, calls `ISerializer<T>.Deserialize`

The adapter is a thin bridge — `IRestSerializer` and `ISerializer<T>` remain independent contracts.

## Content Type

`RestSerializerAdapter<T>` does not set a `ContentType`. Wrap it or set the content type at the HTTP layer to match the backend format (e.g. `application/x-memorypack`, `application/x-msgpack`, `application/json`).
